YiKe / Material 3 Mobile Prototype

忆刻手机 App UI/UX 原型

这一版在原有首页、复习和内容管理流基础上，补齐了 `feature-extension-priority.md` 中 P0 的 统计、搜索、今日预览与熟练度可视化草图，方便继续往 Android Compose 落地。

主色 / Primary

用于开始复习、关键确认按钮和流程进度，保持任务导向感。

容器色 / Primary Container

承接今日任务、轻状态提示与顶部概览，减少大面积纯色压迫。

强调色 / Tertiary

用于 FAB、风险提示与高注意力动作，让重要路径更容易被看到。

表面层 / Surface

大面积背景保持浅色高对比，更适合长时间阅读题面、答案和分析数据。

信息架构

首页、卡组、设置三段式底部导航承接日常入口；复习流作为独立沉浸式页面； P0 扩展以“独立页 + 轻入口”的方式并入现有结构，避免首页和列表页承担过多复杂职责。

01 / Home

首页

任务总览、开始复习、P0 入口聚合

首页仍然只回答“今天该做什么”，但新增了今日预览、复习统计和问题检索入口， 让扩展能力被看见，同时不破坏主路径。

单独打开 看下一页

02 / Today Preview

今日复习预览

按卡组汇总任务、预计耗时、开始前预判负荷

预览页承担的是“开始前建立预期”，因此先给总量、时间和重点卡组，再展开到题目所属卡片， 帮助用户决定是直接开刷还是先处理困难题。

单独打开 看下一页

03 / Analytics

复习统计页

连续学习、评分分布、遗忘率、平均响应时间

统计页优先展示节奏指标和结论，不追求复杂图表，而是用更适合手机端的 KPI 卡片和分布条， 让用户能很快看懂最近学习状态。

单独打开 看下一页

04 / Search

问题搜索与筛选

全文搜索、标签筛选、状态筛选、熟练度标签

搜索页把题库筛选和熟练度标签合在一起，是为了让“找到问题”和“判断是否该复习”成为同一步动作， 也方便后续扩展专项复习入口。

单独打开 看下一页

05 / Review

复习页

问题聚焦、显示答案、四档评分

复习页本质上是沉浸式任务页，因此让题面和评分区拥有最大的纵向空间， 减少用户把注意力浪费在导航和装饰上。

单独打开 看下一页

06 / Decks

卡组列表

主题聚合、新建卡组、今日到期信息

卡组页承担的是“整理学习主题”的职责，所以列表项中同时显示规模和今日到期量， 让管理动作与复习优先级能够一起被看见。

单独打开 看下一页

07 / Cards

卡片列表

章节级内容管理、题库检索入口、熟练度概览

卡片层相当于主题下的章节目录，新增熟练度概览和题库检索入口之后， 用户不必进入编辑页也能先判断“哪张卡最值得处理”。

单独打开 看下一页

08 / Editor

问题编辑

卡片信息 + 多问题编辑，贴近移动端表单习惯

编辑页优先保证输入效率，因此采用长页面单列结构，而不是在移动端硬塞双栏， 让用户能连续录入多个问题而不频繁切换焦点。

单独打开 看下一页

09 / Settings

设置

提醒、版本信息、备份恢复入口

设置页承接的是低频但重要的全局能力，因此内容分组更克制， 重点放在提醒与数据安全，而不是堆叠大量系统项。

单独打开 看下一页

10 / Backup

备份与恢复

高风险操作清晰分层，减少误触

这类页面最怕“看起来轻松，实际上危险”，因此把风险说明抬到页面顶部， 并给导出与恢复两种动作明显区分的按钮层级。
