# 忆刻学习指南

## 科学学习方法，让记忆更牢固

---

## 为什么你需要「忆刻」？

学习新知识时，你是否有过这样的经历：

- 当时学得很明白，过几天却忘得一干二净？
- 翻开书感觉「我看过」，合上书却说不出所以然？
- 花了大量时间重复复习，但效果却不尽如人意？

这些问题的根源在于：**大多数人的学习方式是被动的、低效的。**

忆刻基于认知科学的核心原理设计，帮你用更科学的方式记住更多内容。

---

## 原理一：艾宾浩斯遗忘曲线

### 记忆的天然敌人

1885 年，德国心理学家赫尔曼·艾宾浩斯（Hermann Ebbinghaus）通过大量实验发现了一条规律：**新学的知识会随着时间快速遗忘。**

他用无意义音节进行测试，得到了一条经典的遗忘曲线：

| 时间间隔 | 遗忘率 |
|---------|--------|
| 20分钟后 | 约42% |
| 1天后 | 约66% |
| 6天后 | 约75% |
| 一个月后 | 约79% |

这意味着：**如果不主动复习，你刚学完的内容会在一个月内流失近 80%！**

### 复习的「黄金节点」

艾宾浩斯还发现一个关键规律：**在即将遗忘时进行复习，效果最好。**

因为这时大脑会经历一次「主动检索」的过程——你必须努力回忆内容，这个过程本身就在强化记忆。

基于这个原理，忆刻会按“学习日（D0）+ 间隔天数”的口径，为你生成默认复习计划（最多 10 轮）：

| 复习轮次 | 学习后第几天（D0 为学习日） |
|---------|----------------------------|
| 第1次复习 | 第1天 |
| 第2次复习 | 第2天 |
| 第3次复习 | 第4天 |
| 第4次复习 | 第7天 |
| 第5次复习 | 第15天 |
| 第6次复习 | 第30天 |
| 第7次复习 | 第60天 |
| 第8次复习 | 第90天 |
| 第9次复习 | 第120天 |
| 第10次复习 | 第180天 |

> 说明：
> - 以上为默认间隔口径，具体轮次可在“复习计划预览”中启用/禁用或调整间隔天数。
> - 忆刻的所有日期计算均以“学习日按年月日归一化”为基准（避免时分秒导致跨天偏移）。

## 原理二：主动检索

### 被动复习 vs 主动检索

很多人习惯的复习方式是：**翻开书看一遍。**

这其实是被动阅读，大脑只是在「识别」信息，而不是「回忆」信息。研究表明，这种方式的记忆留存率极低。

**主动检索**则是另一种方式：

- 盖上书，试图用自己的话复述内容
- 做练习题，尝试写出答案
- 给自己出题，检验掌握程度

### 在忆刻中如何实践主动检索？

当你打开忆刻，看到一条复习任务时：

1. **先回忆，再查看**
   - 看到任务标题后，先停下来想想：「这个知识点讲的是什么？」
   - 努力在脑海中检索相关信息
   - 想清楚后，再查看你的笔记或资料对照

2. **对“完成/跳过”保持诚实**
   - 如果你只是机械地点“完成”，那只是在走形式
   - 真正有效的复习是：**不借助任何提示，也能复述出核心内容**
   - 做不到时，可以先不标记完成，回到笔记补齐理解后再来复习

3. **用自己的话总结**
   - 复习时，尝试把学到的内容用自己的语言表述一遍
   - 这比单纯看一遍笔记有效得多

## 原理三：必要难度法则

### 为什么学习不能太「舒服」？

认知心理学家比约克（Robert Bjork）提出一个重要概念：**必要难度（Desirable Difficulties）**。

意思是：**学习过程中适度的困难，反而会增强长期记忆。**

这似乎与直觉相悖——我们总希望学习过程越轻松越好。但研究表明，如果学习时感觉太「顺」，知识往往学得快、忘得也快。

### 四种必要难度在忆刻中的应用

#### 1. 时间间隔（Spacing Effect）

| 传统方式 | 忆刻方式 |
|---------|---------|
| 集中在短期内复习多次 | 分散在更长周期的多个节点复习 |
| 当时记住，很快忘记 | 间隔更科学，长期记忆更牢 |

**忆刻的做法**：自动为你安排分散的复习时间，让每次复习都像「重新学习」一样需要检索。

#### 2. 检索练习（Retrieval Practice）

| 传统方式 | 忆刻方式 |
|---------|---------|
| 翻开书看一遍 | 先回忆，再对照笔记 |
| 识别信息为主 | 回忆信息为主 |

#### 3. 交错学习（Interleaving）

在同一时间段复习不同主题的内容，会让大脑更擅长区分与迁移。

#### 4. 生成效应（Generation Effect）

自己“生成”出来的答案，比看来的更牢固。你在忆刻中用自己的语言写下的描述/子任务，本身就是在强化记忆。

---

## 最后的小建议

- 把标题写成“可检索”的形式（例如：把“集合”改成“Java 集合框架：List/Set/Map 的区别”）
- 描述/子任务尽量写成“能自测”的问题（例如：用问句、对比、例题）
- 每次复习尽量用 2-5 分钟完成一次“回忆→验证→修正”
