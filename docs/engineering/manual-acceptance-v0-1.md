# 首版手动验收清单（v0.1）

## 1. 目的

本清单用于在自动化测试之外，确认忆刻首版的关键主路径在真实设备上可理解、可恢复、可长期自用。

---

## 2. 内容管理

- [ ] 新建卡组后可在列表看到该卡组
- [ ] 进入卡组后可新建卡片并进入问题编辑页
- [ ] 保存包含空答案的问题后，重新进入仍能看到题面与空答案
- [ ] 删除卡片时，相关问题与复习记录被一并清理
- [ ] 归档卡组或卡片后，默认列表与首页待复习不再展示对应内容

---

## 3. 首页与复习流

- [ ] 首页有待复习内容时显示卡片数、问题数和“开始复习”
- [ ] 首页无待复习内容时显示“今日暂无待复习”和“进入卡组”
- [ ] 进入复习页后默认只显示题面，不显示评分按钮
- [ ] 点击“显示答案”后出现答案；空答案时显示“无答案”
- [ ] 提交评分后自动进入下一题；最后一题后进入“本卡完成”
- [ ] 未完成本卡时返回，能看到“未评分的问题不会计入完成”的退出确认
- [ ] 评分失败时页面仍停留在当前题，可再次点击评分重试

---

## 4. 每日提醒

- [ ] 设置页可开启/关闭每日提醒
- [ ] 修改提醒时间后重新打开设置页仍显示新时间
- [ ] Android 13+ 首次开启提醒时会请求通知权限
- [ ] 拒绝通知权限后仍可保存提醒，并看到“通知权限未开启，提醒可能无法显示”
- [ ] 有待复习内容时，到点后收到每日复习通知
- [ ] 无待复习内容时，到点后不发送通知
- [ ] 重启设备或修改系统时区后，提醒仍会按本地固定时间继续工作

---

## 5. 备份与恢复

- [ ] 备份页常驻显示“恢复会覆盖当前本地全部数据”的风险提示
- [ ] 导出空数据集时可成功生成 JSON 文件
- [ ] 导出非空数据集后，文件包含 `app`、`settings`、`decks`、`cards`、`questions`、`reviewRecords`
- [ ] 选择恢复文件后，系统先弹出二次确认，再执行覆盖恢复
- [ ] 合法备份恢复成功后，卡组、卡片、问题、复习记录与提醒设置全部恢复
- [ ] 非法或不兼容备份文件被拒绝，并提示“备份文件无效或版本不兼容”
- [ ] 恢复失败时提示“当前数据未被修改”

---

## 6. Debug 构建专用验收

- [ ] debug 构建首页底部显示“调试工具”按钮
- [ ] 点击“生成随机数据”后，首页待复习统计与卡组列表可看到新增测试数据
- [ ] release 构建首页不显示“调试工具”，且无法进入 `debug` 路由

---

## 7. 回归建议

- [ ] 运行 `./gradlew.bat testDebugUnitTest`
- [ ] 运行 `./gradlew.bat assembleDebugAndroidTest`
- [ ] 在至少一台 Android 13+ 设备上完成提醒与权限相关验收
- [ ] 在至少一台存在历史数据的设备上完成导出 -> 清空 -> 恢复 -> 再次复习的完整回归
