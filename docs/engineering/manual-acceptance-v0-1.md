# 首版手动验收清单（v0.1）

## 1. 目的

本清单用于在自动化测试之外，确认忆刻首版的关键主路径在真实设备上可理解、可恢复、可长期自用。

## 1.1 历史验证记录（2026-03-15）

- 该记录仅代表 2026-03-15 当时的设备侧回归结果，不代表当前 HEAD 已自动满足以下全部手动项。
- 当时已执行 `.\gradlew.bat testDebugUnitTest`
- 当时已执行 `.\gradlew.bat connectedDebugAndroidTest`
- 当时已通过连接设备 `RMX3852 / Android 16` 完成自动化 UI 与集成测试运行

## 1.2 当前使用方式（2026-03-17）

- 当前建议先执行 `.\scripts\verify-testing.ps1`
- 若本轮改动涉及网页后台富后台，再执行 `.\scripts\verify-webconsole.ps1`
- 若本轮改动涉及设备行为，再执行 `.\scripts\verify-testing.ps1 -Connected`
- 以下手动验收项仍应由开发者或产品在真实点击路径中逐项勾选，不应由自动化结果直接替代

## 2. 内容管理

- [ ] 新建卡组后可在列表看到该卡组
- [ ] 进入卡组后可新建卡片并进入问题编辑页
- [ ] 保存包含空答案的问题后，重新进入仍能看到题面与空答案
- [ ] 删除卡片时，相关问题与复习记录被一并清理
- [ ] 归档卡组或卡片后，默认列表与首页待复习不再展示对应内容

---

## 3. 首页与复习流

- [ ] 首页一级导航显示图标、文本和清晰选中态，首页首屏只保留一个正式复习主 CTA
- [ ] 首页有待复习内容时显示卡片数、问题数和“开始今日复习”
- [ ] 首页无待复习内容时显示“今日暂无待复习”和“进入卡组”
- [ ] 进入复习页后默认只显示题面，不显示评分按钮
- [ ] 点击“显示答案”后出现答案；空答案时显示“无答案”，并同步出现“等待评分”节奏提示
- [ ] 展开答案后四档评分都显示对应的下次复习间隔暗示与阶段变化提示
- [ ] 提交评分后自动进入下一题；最后一题后进入“本卡完成”，并显示评分分布与耗时摘要
- [ ] 未完成本卡时返回，能看到“未评分的问题不会计入完成”的退出确认
- [ ] 评分失败时页面仍停留在当前题，可再次点击评分重试
- [ ] 今日全部待复习清空时，复习队列页显示“今日复习已经完成”，并提供“进入自由练习 / 返回首页”

---

## 3.1 练习流补充验收

- [ ] 练习设置页首屏持续显示卡组 / 卡片 / 题目三级范围摘要和预计题量
- [ ] 当前练习范围命中 0 题时，页面直接提示空范围并允许恢复题目全选
- [ ] 自由练习会话顶部明确标明“不会影响正式复习计划”，并显示当前题号与总题量进度条
- [ ] 练习最后一题完成后进入“本轮自由练习已完成”，展示题量、耗时和返回路径

---

## 4. 每日提醒

- [ ] 设置页可开启/关闭每日提醒
- [ ] 修改提醒时间后重新打开设置页仍显示新时间
- [ ] Android 13+ 首次开启提醒时会请求通知权限
- [ ] 拒绝通知权限后仍可保存提醒，并看到“通知权限未开启，提醒可能无法显示”
- [ ] 有待复习内容时，到点后收到每日复习通知
- [ ] 无待复习内容时，到点后不发送通知
- [ ] 重启设备或修改系统时区后，提醒仍会按本地固定时间继续工作

---

## 5. 备份与恢复

- [ ] 备份页常驻显示“恢复会覆盖当前本地全部数据”的风险提示
- [ ] 导出空数据集时可成功生成 JSON 文件
- [ ] 导出非空数据集后，文件包含 `app`、`settings`、`decks`、`cards`、`questions`、`reviewRecords`
- [ ] 选择恢复文件后，系统先弹出二次确认，再执行覆盖恢复
- [ ] 合法备份恢复成功后，卡组、卡片、问题、复习记录与提醒设置全部恢复
- [ ] 非法或不兼容备份文件被拒绝，并提示“备份文件无效或版本不兼容”
- [ ] 恢复失败时提示“当前数据未被修改”

---

## 6. Debug 构建专用验收

- [ ] debug 构建首页底部显示“调试工具”按钮
- [ ] 点击“生成随机数据”后，首页待复习统计与卡组列表可看到新增测试数据
- [ ] 生成随机数据后进入局域网同步预览，可看到本机新增的调试数据变更计数
- [ ] 在一台 debug 设备点击“清除数据”并完成同步后，另一台 debug 设备上的对应调试数据也被删除
- [ ] release 构建首页不显示“调试工具”，且无法进入 `debug` 路由

---

## 7. 网页后台富后台

- [ ] 电脑浏览器通过 `/login` 登录后，能看到按“学习核心 / 内容维护 / 系统管理”分组的导航、顶部状态芯片、上下文栏和当前主工作区
- [ ] 手机浏览器访问同一后台时，顶部菜单入口显示为高辨识度“菜单导航”，折叠菜单展开后仍能感知当前工作区位置
- [ ] 切换 `/overview`、`/study`、`/content`、`/search`、`/analytics`、`/settings`、`/backup` 时，壳层主标题与浏览器标题同步更新
- [ ] 内容管理工作区可按卡组 -> 卡片 -> 问题逐层下钻，右侧工作台会同步展示当前上下文
- [ ] 在内容工作台中编辑卡组、卡片或问题后，原 drill-down 路径和 URL 查询参数仍被保留，不会退回无上下文列表
- [ ] URL 查询参数包含 `deckId=null`、`cardId=null` 或空值时，刷新后不会破坏内容工作台恢复
- [ ] 从内容工作台直接发起“练当前卡组 / 卡片 / 问题”后，会进入学习工作区并保留来源标签
- [ ] 内容工作台的“查看 / 编辑 / 练习 / 删除”在 deck、card、question 三层级都保持清晰对象语义
- [ ] 归档卡组、归档卡片、删除问题、恢复备份等高风险动作都会先展示对象级确认
- [ ] 内容工作台提供“批量归档本卡组卡片 / 批量删除本卡片问题”入口，并明确说明作用范围
- [ ] 搜索结果可直接发起“练这题”，结束后会返回搜索来源而不是丢失原上下文
- [ ] `/overview`、`/study`、`/content`、`/search`、`/analytics`、`/settings`、`/backup` 都可直接访问，刷新后仍停留在对应页面
- [ ] 浏览器登录后能在学习工作区看到今日待复习卡片数、问题数与当前会话状态
- [ ] 今日复习存在待复习内容时，可从浏览器直接进入首张待复习卡片并看到卡片上下文与题目进度
- [ ] 浏览器端复习默认只显示题面；点击“显示答案”后才出现答案和四档评分按钮
- [ ] 浏览器端完成当前卡片最后一题后，会出现“本卡完成”提示，并可继续下一张或结束会话
- [ ] 自由练习至少选择一个范围后才能开始；若当前范围命中 0 题，会看到可理解的空状态提示
- [ ] 浏览器端自由练习支持显示答案、上一题、下一题与结束练习，且不会改变正式复习统计
- [ ] 在浏览器学习过程中刷新页面，能够恢复到最近一次有效学习题位
- [ ] 搜索、统计、设置和备份恢复工作区都带有统一的顶部反馈条，并能区分空状态、错误状态和高风险提示
- [ ] 备份恢复工作区在选择文件、等待确认、恢复中和恢复完成后，都能看到明确的风险/结果反馈
- [ ] 刷新访问码或停止网页后台服务后，原浏览器学习会话失效，并提示重新登录或重新进入学习入口

---

## 8. 局域网同步

- [ ] 两台设备进入同步页后，彼此都能在候选设备列表中看到对方
- [ ] 首次配对输入相同配对码后，双方都进入可信状态并能继续同步
- [ ] 一侧存在本地变更、另一侧存在远端变更时，同步预览能展示冲突与计数
- [ ] 完成一次同步后，另一台设备可看到新增或更新的数据
- [ ] 退出同步页或断开网络后，设备列表会在合理时间内清空或刷新

---

## 9. 回归建议

- [ ] 运行 `.\scripts\verify-testing.ps1`
- [ ] 如本轮涉及设备行为，运行 `.\scripts\verify-testing.ps1 -Connected`
- [ ] 在至少一台 Android 13+ 设备上完成提醒与权限相关验收
- [ ] 在至少一台存在历史数据的设备上完成导出 -> 清空 -> 恢复 -> 再次复习的完整回归
