## 1. 范围与约束

### 1.1 允许的变更边界

| 层级 | 允许变更 | 不允许变更 |
|------|----------|------------|
| **Data 层** | 新增 DAO 查询方法、扩展现有仓储实现 | 修改表结构、变更 SchemaVersion、新增 Drift 表 |
| **Domain 层** | 新增实体（纯 Dart）、新增/扩展 UseCase、仓储接口 | 禁止破坏既有字段语义/导致旧数据无法解析（允许以向后兼容方式新增可选字段） |
| **Presentation 层** | 新增 Provider、新增/修改页面、新增组件、新增路由 | - |
| **Infrastructure 层** | 新增服务实现、扩展通知/同步逻辑 | - |
| **配置存储** | 使用现有 settings 表（UI 偏好） | 新增 Drift 表存储配置、禁止引入 SharedPreferences |

**关键约束**：所有可持久化配置（目标设定、主题色、通知偏好）必须写入现有 `settings` 表，禁止新增数据库表或引入 SharedPreferences。

## 3. 详细改进方向

### 3.1 统计增强（高优先级 P0）

#### 3.1.1 趋势图表

**描述**：在统计页面增加完成率趋势折线图，支持周/月/年视图切换。

**统计口径定义**

| 口径项 | 规则 | 与现有实现对照 |
|--------|------|----------------|
| 完成数 | 状态为 `done` 的任务数 | 与 `get_statistics_usecase.dart` 一致 |
| 跳过数 | 状态为 `skipped` 的任务数（**不计入**完成率） | 与现有口径一致 |
| 总数 | 状态为 `done` + `pending` 的任务数（skipped 不计入） | 与现有实现一致 |
| 完成率 | `done / 总数 * 100%` | 与现有实现一致 |
| 时区 | 本地时区，以 `00:00` 为日界线 | - |
| 缺失日期 | 补 0（无任务日期显示为 0） | - |
| 周起始日 | **周一** | - |
| 归因日期 | 按 `scheduledDate`（计划复习日） | 与现有统计/连续打卡实现一致（review_task_dao.dart） |

**交互细节**

| 元素 | 规格 |
|------|------|
| 视图切换 | Tab 切换：本周 / 本月 / 本年 |
| 图表类型 | 折线图（fl_chart LineChart，项目已存在） |
| 数据点 | 本周/本月：每日一个点；本年：按月聚合（12 个点） |
| tooltip 触发 | 桌面端：hover 显示；移动端：点击/长按显示 |
| tooltip 文案 | 本周/本月：`"{日期}: {完成数}/{总数} ({完成率}%)"`；本年：`"{月份}: {月完成数}/{月总数} ({完成率}%)"` |
| 边界文案 | 总数 = 0 时显示"无任务"；done = 0 且总数 > 0 时显示"0/{总数} (0%)" |
| Y 轴范围 | 0% ~ 100%（固定） |
| 加载状态 | 数据加载中显示骨架屏，<200ms 不显示避免闪烁 |
| 性能目标 | 渲染 30 天数据 <16ms（60fps） |

**关键文件**

- `lib/presentation/providers/statistics_provider.dart`
- `lib/presentation/widgets/statistics_content.dart`

**验收标准**

- Given: 用户打开统计页面
- When: 切换到趋势图表 Tab
- Then: 显示近 N 天的完成率折线图，tooltip 正确显示数据

#### 3.1.3 目标设定

**描述**：允许用户设置学习目标，并在统计页面展示进度。

**目标类型**

| 类型 | 描述 | 数据来源 |
|------|------|----------|
| 每日完成数 | 每天完成 N 个任务 | 统计 `done` 任务数 |
| 连续打卡 | 连续完成学习的天数 | 统计 `consecutiveCompletedDays` |
| 本周完成率 | 本周完成率 >= N% | 统计本周 `done / (done + pending)` |

**约束**

- 目标值存储到现有 `settings` 表（通过 settings DAO，key: `goal_daily` / `goal_streak` / `goal_weekly_rate`）
- 进度**完全由现有任务数据派生**，不新增数据库字段
- 支持多目标（上限 3 个）
- 参考现有 settings 表结构：`lib/data/database/tables/settings_table.dart`

**交互细节**

| 元素 | 规格 |
|------|------|
| 设置入口 | 设置页 → 学习目标 |
| 展示位置 | 统计详情 Sheet 顶部 |
| 进度显示 | 环形进度条 + 百分比 + 目标值 |
| 达成反馈 | 达成目标时显示庆祝动画（复用 completion_animation） |

**关键文件**

- `lib/presentation/providers/goal_provider.dart`（新建）
- `lib/presentation/pages/settings/goal_settings_page.dart`（新建）

**验收标准**

- Given: 用户设置每日完成 10 个任务
- When: 当日完成 10 个任务后查看统计
- Then: 显示"目标达成"状态和庆祝动画

**验收标准**

- Given: 页面加载耗时 300ms
- When: 用户进入页面
- Then: 显示骨架屏，加载完成后平滑过渡到内容

### 3.3 性能优化

#### 3.3.1 Provider 精准订阅扩展

**描述**：将 `select()` 精准订阅模式推广到更多 Provider。

**评估准则**

| 指标 | 阈值 | 行动 |
|------|------|------|
| Provider 重建次数 | >10次/秒 | 需要 select |
| 平均帧耗时 | >16ms | 需要分析依赖链 |
| 关联组件数 | >5个 | 考虑拆分 |

**已有示例**（需推广）

- `todayProgressProvider` 使用 `select((s) => (s.isLoading, s.completedCount, s.totalCount))`
- `taskHubProvider` 状态分离：`timelineRows` vs `expandedTaskIds`

**验证方式**

- 使用 Flutter DevTools 对比优化前后 rebuild 次数

**验收标准**

- Given: Provider 监听的字段未变化
- When: 其他无关字段变化
- Then: 依赖该 Provider 的 Widget 不重建

---

#### 3.3.2 图片资源预加载

**描述**：应用启动后预加载关键资源，减少首帧渲染抖动。

**预加载清单**

| 资源类型 | 预加载时机 | 方式 |
|----------|------------|------|
| 首页顶部图标 | 首帧后 `addPostFrameCallback` | `precacheImage` |
| 自定义字体 | 应用启动 | `FontLoader.load` |
| 常用 SVG | 首帧后延迟 500ms | `precachePicture` |

**约束**

- 预加载不能拉长启动时间（冷启动 <2s）
- 内存敏感设备跳过预加载（检测低内存）

**验收标准**

- Given: 应用冷启动
- When: 首页渲染
- Then: 关键图标无加载抖动

---

#### 3.3.3 列表 keepAlive（可选增强）

**描述**：使用技术手段保持列表滚动位置和状态。

**当前痛点定义**

| 痛点 | 场景 | 解决策略 |
|------|------|----------|
| 滚动位置丢失 | 切换底部导航 tab | `PageStorageKey` |
| 重新请求数据 | 切换 tab 后返回 | `AutomaticKeepAliveClientMixin`（可选） |
| 展开状态丢失 | 列表滚动后展开的任务 | `PageStorageKey` + Provider 状态保持 |

**内存考量**

- 使用 `AutomaticKeepAliveClientMixin` 需谨慎，确保最多保持 2 个页面
- 监听内存压力（`WidgetsBindingObserver.didHaveMemoryWarning`）后减少缓存/关闭 keepAlive

**验收标准**

- Given: 用户在任务中心滚动一段距离后切换到首页
- When: 用户切回任务中心
- Then: 滚动位置保持

---

#### 3.3.4 延迟加载非关键组件

**描述**：对非首屏组件使用延迟渲染。

**适用场景**

| 组件 | 延迟触发 | 延迟时间 |
|------|----------|----------|
| 首页底部统计栏 | 滚动到可见区域 | 首帧后 300ms |
| 日历页面统计详情 | 点击展开 | 立即 |
| 任务中心底部推荐 | 滚动到底部 80% | 立即 |

**实现方式**

- 首选：滚动阈值触发（更轻量）
- 备选：`VisibilityDetector`（需新增依赖）

**验收标准**

- Given: 首页数据加载完成
- When: 用户未滚动到底部
- Then: 底部统计栏不阻塞首屏渲染

| 元素 | 规格 |
|------|------|
| 触发时机 | 完成/跳过后**立即**显示 |
| 持续时间 | 5 秒（可配置） |
| 撤销范围 | 仅限最近一次操作 |
| 连续操作 | 新操作覆盖前一个 Snackbar |

**边界处理**

| 场景 | 行为 |
|------|------|
| 用户在 5s 内撤销 | 恢复任务状态，隐藏 Snackbar |
| 5s 后无操作 | 状态不可逆（操作已立即生效，5s 只是撤销窗口期） |
| 连续快速操作 | 新操作覆盖前一个 Snackbar |

**验收标准**

- Given: 用户完成一个任务
- When: 操作成功
- Then: 显示 Snackbar "任务已完成"，5s 内可撤销

#### 3.6.2 精准定时通知

**描述**：使用 `zonedSchedule` 实现精确时间提醒。

**依赖**

- 需要 `timezone` 包处理时区计算
- 需在应用启动时初始化时区数据

**系统限制**

| 平台 | 精度 | 降级行为 |
|------|------|----------|
| Android | 分钟级 | 允许 ±5 分钟延迟 |
| iOS | 分钟级 | 系统调度，允许延迟 |

**验收标准**

- Given: 用户设置提醒时间为 20:00
- When: 系统时间到达 20:00
- Then: 通知触发（允许 ±5 分钟误差）

#### 3.6.3 多通知类型

**去重策略**

**验收标准**

- Given: 用户连续打卡 7 天

- Then: 收到"连续打卡 7 天"通知
