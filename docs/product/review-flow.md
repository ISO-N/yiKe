# 复习流程定义（v0.1）

## 1. 文档目的

本文件用于定义忆刻 v0.1 的核心用户流程与页面状态流转，确保产品、交互与开发实现对“复习是如何发生的”保持一致理解。

---

## 2. 核心流程概览

忆刻的主路径应尽可能短：

**首页 -> 开始复习 -> 进入卡片 -> 逐题回忆 -> 显示答案 -> 单题评分 -> 完成本卡 -> 返回首页/下一卡**

设计目标：

- 打开即知道今天要做什么
- 尽快进入复习状态
- 尽量减少跳转与多余信息

---

## 3. 首页流程

### 3.1 首页主要信息

首页默认展示：

- 今日待复习卡片数
- 今日待复习问题数
- “开始复习”按钮
- “自由练习”次入口
- 简要卡组入口
- 若已完成，则展示“今日已完成”状态

### 3.2 首页状态

#### 状态 A：有待复习

展示：

- 今日待复习数量
- 立即开始复习按钮

#### 状态 B：无待复习

展示：

- 今日暂无待复习
- 可以进入卡组管理
- 可以创建新内容

#### 状态 C：加载失败

展示：

- 错误提示
- 重试按钮

---

## 4. 开始复习后的选题逻辑

### 4.1 进入入口

用户点击“开始复习”后，系统进入待复习卡片队列。

### 4.2 队列内容

队列中的每一项是“卡片”，但卡片内部只包含当前到期的问题集合。

### 4.3 卡片排序建议

第一版可采用简单规则：

- 按卡组排序 + 卡片创建顺序
- 或按最早 dueAt 升序

第一版不必过度优化排序策略。

---

## 5. 卡片复习流程

### 5.1 进入卡片时的页面结构

建议页面包含：

- 所属卡组名称
- 卡片标题
- 当前问题序号（如 1/3）
- 当前问题内容
- “显示答案”按钮
- 评分按钮区

### 5.2 初始状态

进入某个问题时：

- 默认只展示问题
- 不直接显示答案
- 评分按钮默认隐藏或禁用

### 5.3 点击“显示答案”后

- 展示该问题的答案
- 若答案为空，则显示“无答案”
- 评分按钮可见

### 5.4 完成评分后

系统执行：

1. 记录 ReviewRecord
2. 更新 Question 的 stageIndex
3. 更新 Question 的 dueAt
4. 更新 lastReviewedAt
5. 跳转到本卡片下一道到期问题

---

## 6. 逐题切换规则

第一版采用 **逐题切换**，不在一页中同时展示所有问题。

原因：

- 更聚焦
- 评分动作更明确
- 更符合“主动回忆”的节奏
- 更便于后续做动画和状态控制

### 6.1 单题完成后

- 立即进入下一题
- 不要求用户手动返回问题列表

### 6.2 当前卡片题目全部完成后

- 显示“本卡完成”状态
- 用户可选择继续下一张卡、查看统计，或返回首页

---

## 7. 今日完成规则

### 7.1 问题级别

一个问题只要完成一次评分，就视为本轮已处理完成。

### 7.2 卡片级别

当本次进入该卡片时加载出的所有到期问题都完成评分后，该卡片本轮完成。

### 7.3 当天是否再次出现

不会。

即使评分为“完全不会”，问题也不会在当天再次刷回。

---

## 8. 评分动作细节

### 8.1 显示条件

只有在用户点击“显示答案”后，评分按钮才出现。

### 8.2 四个评分按钮

- 完全不会
- 有印象
- 基本会
- 很轻松

### 8.3 评分后的即时反馈

第一版建议最小反馈：

- 可选 Toast / Snackbar：如“已记录”
- 或直接无提示，顺滑跳下一题

优先保持流畅，不打断节奏。

---

## 9. 空答案处理

若当前问题没有填写答案：

- 点击“显示答案”后展示文本：`无答案`
- 仍允许继续评分
- 不视为异常

这样能兼容“先录问题、后补答案”的自用场景。

---

## 10. 中途退出处理

### 10.1 用户中途返回首页

若用户尚未对当前问题评分：

- 本题不计入已完成
- 下次仍会出现在今日待复习中

### 10.2 用户已评分后再退出

- 已评分问题已持久化
- 下次从剩余 due 问题继续

第一版建议采用“评分后立即落库”的策略，避免进度丢失。

---

## 11. 卡组与内容管理流程

虽然复习是主路径，但内容管理仍需清晰。

### 11.1 卡组页

支持：

- 查看卡组列表
- 创建卡组
- 进入某卡组

### 11.2 卡片页

支持：

- 查看该卡组下卡片列表
- 创建卡片
- 编辑卡片标题
- 删除或归档卡片

### 11.3 问题编辑页

支持：

- 在卡片下添加多个问题
- 为每个问题填写答案（可空）
- 编辑问题与答案
- 删除问题

---

## 12. 复习完成后的路径

当所有今日待复习卡片完成后：

- 提供统一收尾引导
- 用户可选择继续复习、查看统计或返回首页
- 首页显示“今日已完成”
- 可提供轻量完成反馈，如：
  - 今日已完成 12 个问题
  - 明日继续

第一版不必加入复杂的庆祝动画。

---

## 13. 异常流程

### 13.1 卡片无到期问题

若用户从管理页进入某卡片，而当前没有待复习问题：

- 正常展示内容管理视图
- 不进入复习态

### 13.2 问题数据缺失

若问题存在但题面为空（理论上不应发生）：

- 不允许进入复习
- 应在数据层拦截

### 13.3 评分落库失败

- 显示错误提示
- 当前题不应被跳过
- 用户可重试

---

## 14. 流程图（文字版）

```text
首页
  -> 点击“开始复习”
    -> 加载今日待复习卡片队列
      -> 进入第 1 张卡片
        -> 展示第 1 道到期问题
          -> 点击“显示答案”
            -> 展示答案 / 无答案
                -> 点击四档评分之一
                -> 写入复习记录
                -> 更新问题调度
                -> 若还有下一题 -> 切到下一题
                -> 若本卡完成 -> 继续复习 / 查看统计 / 返回首页
```

---

## 15. 页面状态建议

### 首页状态

- Loading
- HasDueItems
- EmptyToday
- Error

### 复习页状态

- Loading
- QuestionOnly
- AnswerVisible
- Saving
- SaveError
- CardCompleted
- SessionCompleted

---

## 16. 第一版交互取舍

建议第一版不要加入：

- 左右滑手势评分
- 长按快捷评分
- 音效
- 翻牌动画
- 多题并排
- 自动朗读

这些都可能干扰基本闭环的稳定实现。

---

## 17. 结论

忆刻 v0.1 的复习流程应遵循一条清晰原则：

**让用户尽快进入“想—看—评”的节奏，而不是停留在界面操作本身。**

只要这条主路径流畅，产品的核心价值就成立了。

---

## 18. 练习模式并行流程

### 18.1 目标

练习模式用于承接“我现在想主动巩固某一组内容”的场景，它与正式复习并行存在，但不替代“今日到期复习”的主路径。

### 18.2 入口位置

第一版练习模式从以下入口进入：

- 首页 Hero 区次入口“自由练习”
- 卡组列表项“练习卡组”
- 卡片列表页顶部“自由练习”与卡片项“练习本卡”
- 搜索页“练习当前结果”与结果项“练习这题”

这些入口都必须明确表达“练习”语义，不能与“开始复习”主按钮混淆。

### 18.3 设置与会话流程

练习模式采用两段式流程：

```text
任意练习入口
  -> 练习设置页
    -> 选择卡组范围
    -> 选择卡片范围
    -> 可选：题目级手选
    -> 选择顺序 / 随机
      -> 进入练习会话
        -> 展示题面
        -> 用户点击“显示答案”
        -> 上一题 / 下一题 / 结束练习
```

### 18.4 练习模式边界

练习模式必须固定遵守以下边界：

- 不写入 `ReviewRecord`
- 不更新 `Question.stageIndex`
- 不更新 `Question.dueAt`
- 不更新 `Question.lastReviewedAt`
- 不更新 `Question.reviewCount`
- 不更新 `Question.lapseCount`
- 不改变首页今日待复习统计
- 不触发提醒重算

### 18.5 练习页节奏

练习页继续复用“先回忆、再看答案”的节奏，但去掉正式评分语义：

- 默认只展示题面
- 点击“显示答案”后才展开答案
- 若答案为空，显示“无答案”
- 提供“上一题 / 下一题 / 结束练习”
- 不展示 `AGAIN / HARD / GOOD / EASY`

### 18.6 顺序与随机规则

- 顺序模式必须使用稳定且可重复的题目顺序
- 随机模式必须在会话开始时只洗牌一次
- 同一会话内的上一题、下一题与进程重建恢复都必须沿用同一顺序

### 18.7 空范围反馈

当当前选择范围命中 0 题时：

- 设置页必须显示可理解的空状态
- 用户必须能够返回调整范围
- 系统不得直接进入空白练习页
