# 发布与版本管理（v0.1）

## 1. 文档目的

本文件用于统一忆刻 v0.1 的版本概念、版本号策略、数据库与备份格式版本管理方式，以及首版发布前后的基本流程。

---

## 2. 为什么需要版本文档

忆刻虽然是离线优先应用，但仍然涉及多种“版本”：

- 应用版本
- 数据库结构版本
- 备份文件格式版本
- 局域网同步协议版本

如果这些版本没有统一管理，后续会出现：

- 升级后数据兼容不清楚
- 导入导出行为不可预期
- 问题排查难以定位

---

## 3. 版本概念定义

### 3.1 应用版本

对应 Android 的：

- `versionCode`
- `versionName`

作用：

- 标识安装包版本
- 用于发布与升级判断

### 3.2 数据结构版本

对应：

- Room 数据库 schema version
- DataStore 中保存的 `schemaVersion`

作用：

- 标识本地数据结构是否发生变化

### 3.3 备份格式版本

对应：

- JSON 顶层 `backupVersion`

作用：

- 标识导入导出文件结构

### 3.4 局域网同步协议版本

对应：

- `syncProtocolVersion`

作用：

- 标识局域网同步的 hello、配对、pull/push/ack 协议代际
- 区分“文件备份兼容”和“设备间同步兼容”

---

## 4. 第一版版本建议

### 4.1 应用版本

- `versionCode = 1`
- `versionName = "0.1.0"`

说明：

- 虽然当前工程模板是 `1.0`，但若项目尚未正式发布，建议切回 `0.1.0` 更符合 MVP 阶段。

### 4.2 数据结构版本

- `schemaVersion = 4`
- Room `version = 4`

### 4.3 备份格式版本

- `backupVersion = 1`

### 4.4 局域网同步协议版本

- `syncProtocolVersion = 2`

---

## 5. 版本递增规则

### 5.1 `versionCode`

每次对外发布都递增。

### 5.2 `versionName`

建议采用 `MAJOR.MINOR.PATCH`：

- `0.1.0`：首个 MVP 版本
- `0.1.1`：仅修复问题
- `0.2.0`：有功能增量但仍未到稳定 1.0
- `1.0.0`：产品能力与稳定性进入正式阶段

### 5.3 `schemaVersion`

仅在本地数据结构发生不兼容变化时递增。

### 5.4 `backupVersion`

仅在导入导出 JSON 结构变化到无法按旧规则解析时递增。

### 5.5 `syncProtocolVersion`

仅在局域网同步协议发生不兼容变化时递增。

---

## 6. 兼容性原则

### 6.1 应用升级

必须保证：

- 新版本应用能读取旧版本本地数据
- 必要时执行数据库迁移
- 协议不兼容时明确阻止旧设备继续同步

### 6.2 备份恢复

第一版只强制保证：

- `backupVersion = 1` 的文件可被同代应用恢复

若后续升级：

- 新版本应用可尝试兼容旧备份版本
- 不要求旧版本应用反向兼容新备份文件

---

## 7. 何时需要迁移

以下情况通常需要提升 `schemaVersion` 并编写 Migration：

- 新增非空字段且无默认值
- 修改字段含义
- 调整主外键关系
- 拆分或合并表

以下情况通常不需要提升 `backupVersion`：

- 新增一个有默认值的可选字段
- 新增仅用于内部展示、不进入备份的数据

---

## 8. 发布前检查清单

首版发布前建议至少确认：

1. `versionCode`、`versionName` 已更新。
2. 数据库版本与文档一致。
3. 备份文件版本与文档一致。
4. 局域网同步协议版本与文档一致。
5. 提醒、备份、恢复、局域网同步已完成手动验证。
5. `keystore.properties` 已在本地配置，且 release keystore 不在仓库内。
6. 已执行一次已签名的 `assembleRelease` 验证 APK 可安装。
7. README 与文档索引已更新。

---

## 9. 发布后记录建议

即使是自用项目，也建议保留最小变更记录。

建议记录：

- 版本号
- 发布时间
- 主要变更
- 是否涉及数据结构升级
- 是否涉及备份格式升级
- 是否涉及同步协议升级

若后续版本节奏形成，可引入 `CHANGELOG.md`。

---

## 10. 结论

忆刻 v0.1 至少需要管理好三种版本：

- 应用版本
- 数据结构版本
- 备份格式版本
- 局域网同步协议版本

把这四件事分开管理，后续迭代就不会因为“版本都叫 1”而混乱。
